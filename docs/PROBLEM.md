# PROBLEM.md — AI认知诊断引擎

## 问题陈述

考试结束后，教师拿到的数据是"谁错了多少分"，而不是"学生到底掌握了什么"。

一个学生在"气候类型判断"题上出错，传统分析只会标记"此题错误"。但真正的问题是：
- 学生是否掌握了等值线判读？（工具能力）
- 学生是否混淆了地中海和温带海洋性气候？（概念理解）
- 学生是否记混了气候类型名称？（记忆准确性）

三种缺失，教学策略完全不同。但现有工具无法区分——因为它们没有诊断模型，只有分数统计。

## 目标用户

**高中地理教师**（MVP阶段），后续扩展到其他学科。

用户画像：带2-3个班（60-100人），每次考试后需要分析错题、制定补课方案，但只有"分数统计"工具可用。

## 核心痛点

1. **知道谁错了，不知道掌握了什么** — 分数统计只能告诉你"张三考了62分"，不能告诉你"张三掌握了哪些知识点、没掌握哪些"
2. **补课凭经验，不精准** — "这次气候题错得多，那就再讲一遍气候"——但可能问题不在气候本身
3. **不知道先补什么** — 5个知识点都薄弱，课时有限，先补哪个效果最大？
4. **补完不知道有没有用** — 补课了，下次考试还是错，不知道是补错了还是补课方式不对

## 现有替代方案及其不足

| 现有方案 | 做了什么 | 做不到什么 |
|---------|---------|-----------|
| Excel成绩统计 | 分数分布、排名 | 不知道掌握了什么 |
| 阅卷系统 | 自动评分、错题统计 | 错题≠知识状态 |
| 教师经验判断 | 凭直觉判断薄弱点 | 主观、不可复现、新手教师做不到 |
| AI分析工具 | 调用LLM分析错题 | 没有自己的诊断模型，结果不可控不可验证 |

## 核心区别：AI原生 vs 调用AI

这是本产品最根本的定位问题。

| | 调用AI的产品 | AI原生产品（我们） |
|---|---|---|
| 诊断逻辑 | LLM推理（黑盒，换模型结果变） | 认知诊断模型CDM（白盒，参数可验证） |
| 知识映射 | LLM分类（prompt驱动） | Q矩阵课标约束初始化+数据联合估计（课标约束→NLP选择→数据验证） |
| 冷启动 | 依赖LLM自由分类 | 课程标准约束候选集，LLM在约束内选择 |
| 知识图谱 | 人工定义+LLM建议 | 课程标准骨架+结构化课件模板+条件独立检验发现 |
| 关系发现 | 统计相关(虚假因果) | 条件独立检验(偏相关，排除混杂) |
| 课件解析 | LLM自由提取(不可控) | 结构化模板解析(确定性)+LLM校验(辅助) |
| 反事实推理 | LLM编一个预测值 | CDM参数推断（数学可证） |
| 反馈学习 | few-shot注入prompt | 模型参数更新 |
| 核心资产 | 没有自己的模型（只有prompt） | CDM参数、Q矩阵、图谱权重 |
| 系统行为 | 被动等待教师操作 | 主动触发：数据变化→自动更新→主动通知 |
| LLM角色 | 引擎（做推理） | 工具（做文本理解），系统有自己的决策触发规则 |

**我们不做模型微调。** 我们构建的是一套AI智能化体系：
- CDM是统计模型，参数从数据中估计（EM算法），不是神经网络微调
- Q矩阵是数据结构，从学生答题数据中联合估计，不是LLM分类结果
- 知识图谱是图结构，从课件+课程标准+数据中构建，不是LLM生成内容
- LLM是工具（文本理解、自然语言生成），不是引擎（诊断推理）
- 系统有自己的决策触发规则（数据变化→自动参数更新→主动通知），不是被动等待教师点击

类比：国际象棋AI有自己的评估函数+搜索算法，不会问GPT"这步怎么走"。推荐系统有自己的协同过滤模型，不会问GPT"推荐什么"。我们的系统有自己的CDM诊断模型，不会问GPT"根因是什么"。

## 成功指标（SMART）

| 指标 | 目标 | 测量方法 |
|------|------|---------|
| 知识状态诊断准确率 | ≥80% | CDM模型拟合指标（AIC/BIC）+ 教师盲评一致性 |
| Q矩阵估计准确率 | ≥90% | 与教师标注的Q矩阵对比 |
| 教学建议采纳率 | ≥60% | 教师在系统中点击"采纳建议"的比例 |
| 诊断效率提升 | ≥3倍 | 对比：系统诊断时间 vs 教师人工分析时间 |
| 补课后改善率 | ≥50% | 采纳建议后，下次考试相关知识点掌握率提升的学生比例 |

## 明确排除（MVP不做）

- ❌ 学生端App（MVP只有教师端Web）
- ❌ 多学科扩展（MVP只做地理）
- ❌ 在线答题/自动批改系统（MVP只导入已批改结果）
- ❌ 自动出题/组卷
- ❌ 学校管理层看板
- ❌ 多版本教材适配（MVP只支持人教版）
- ❌ 基础模型微调（我们用统计模型CDM，不用神经网络微调）
- ❌ 闭环进化（MVP不做自修正，只做诊断+建议+参数更新）

## 核心假设（需要验证）

1. **CDM比LLM推理更可靠** — 统计模型的诊断准确率是否高于LLM的推理？
2. **课件比教材更适合作为知识来源** — 教师的课件是否能更精准地反映实际教学内容？
3. **教师愿意信任模型诊断** — 如果模型给出了参数化的诊断结果，教师是否比只看LLM结论更信任？
4. **量化预测驱动行动** — 给出"补X后Y提升Z%"的量化预测后，教师是否会执行？
