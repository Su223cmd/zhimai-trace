## 1. 产品一句话

**导入课件+已批改作业 → 5-Agent协作构建图谱、诊断状态、追踪轨迹、推荐路径、持续演化 → 量化定位"错在哪、根因是什么、补什么、补了提升多少"**

## 2. 产品定位

**AI知识根因诊断系统** — 以5-Agent协作框架为核心引擎的认知诊断产品。

### AI原生 vs 调用AI

调用AI的产品：LLM是引擎，系统是壳。你问它答，它没有自己的模型、没有自己的参数、没有自己的推理规则。换个LLM，产品逻辑就变了。

AI原生的产品：Agent是引擎，LLM是工具。系统有自己的统计模型（CDM/BKT）、有自己的参数（slip/guess/掌握概率/学习速率）、有自己的推理规则（DINA约束/BKT转移矩阵）、有自己的演化机制（在线EM/参数漂移检测/因果发现）。LLM在约束内选择，不是自由生成。

**5-Agent协作框架是核心**：

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent 协作层                              │
│                                                             │
│  KnowledgeAgent ──→ DiagnosisAgent ──→ TracingAgent         │
│        │                   │                  │              │
│        │                   │                  │              │
│        │                   └──→ TeachingAgent │              │
│        │                        │            │              │
│        └────────────────────────┴────────────┘              │
│                           │                                 │
│                    EvolutionAgent                            │
│                    (增量更新/漂移检测/因果发现/图谱演化)       │
│                           │                                 │
│                    通知 DiagnosisAgent 重新诊断               │
├─────────────────────────────────────────────────────────────┤
│                    模型与数据层                              │
│                                                             │
│  CDM统计模型（DINA/G-DINA）：从数据中估计参数                │
│  BKT知识追踪：时序建模掌握轨迹                               │
│  Q矩阵：课标约束初始化 + NLP/LLM选择 + 数据联合估计          │
│  知识图谱：6层层级 + 前驱/相似/层级关系 + EDC提取            │
├─────────────────────────────────────────────────────────────┤
│                    知识理解层                                │
│                                                             │
│  课件解析：结构化模板确定性提取 + LLM自由模式                 │
│  课标导入：课程标准骨架 + 语义对齐                           │
│  6层层级：学科 → 章节 → 课程 → 知识点 → 考点 → 考法         │
└─────────────────────────────────────────────────────────────┘
```

**LLM的角色：约束内选择，不是自由生成。**
- 知识理解层：LLM在模板结构内校验（模板模式），或在课标约束内提取（自由模式）
- Q矩阵：LLM从课标约束的候选集中选择，不是自由分类
- CDM/BKT：不用LLM，用统计模型
- 教学建议：LLM辅助生成自然语言描述，但预测数值来自模型参数
- Agent决策：基于规则引擎+事件驱动，不依赖LLM判断

### 项目制多学科

每个学科是一个独立项目，共享同一套标准产品。项目之间数据隔离、模型独立、图谱独立。

```
项目A：高中地理
  ├── 独立知识图谱（6层层级）
  ├── 独立Q矩阵
  ├── 独立CDM参数
  └── 独立Agent实例

项目B：高中数学
  ├── 独立知识图谱（6层层级）
  ├── 独立Q矩阵
  ├── 独立CDM参数
  └── 独立Agent实例
```

## 4. 核心流程

### 5-Agent协作流

```
KnowledgeAgent: 课件上传 → 解析知识点 → 构建图谱(6层层级+EDC关系) → 通知DiagnosisAgent
                                                                      ↓
DiagnosisAgent: 收到通知 → 等待答题数据 → Q矩阵生成 → CDM参数估计(EM) → 根因追溯
                                                                      ↓
                                                        ┌──────────────┴──────────────┐
                                                        ↓                              ↓
TracingAgent:                                    收到诊断结果                   TeachingAgent:
                                                 BKT时序建模                   收到诊断结果
                                                 掌握轨迹预测                  学习路径生成
                                                 学习速率估计                  教学活动推荐
                                                 遗忘曲线拟合                  教学建议生成
                                                 掌握时间预测                  效果追踪
                                                        ↓                              ↓
                                                        └──────────────┬──────────────┘
                                                                       ↓
EvolutionAgent:                                          收到新数据/反馈
                                                         在线EM增量更新
                                                         参数漂移检测
                                                         因果发现(偏相关检验)
                                                         图谱演化(新前驱关系)
                                                                       ↓
                                                         通知 DiagnosisAgent 重新诊断
```

### 数据流详解

```
课件(PPT)上传 → KnowledgeAgent解析(模板模式/LLM自由模式)
        ↓
对齐课程标准 → 6层层级构建(学科→章节→课程→知识点→考点→考法)
        ↓
EDC关系提取 → 知识图谱(前驱/相似/层级关系) + 语义对齐
        ↓
作业数据导入(已批改) → DiagnosisAgent: 课标约束+NLP/LLM/手动初始化Q矩阵
        ↓
CDM参数估计(DINA, EM算法) → 每个学生×每个知识点的掌握概率
        ↓
根因追溯 → 沿图谱前驱链 + CDM参数推断根因
        ↓
TracingAgent: BKT时序建模 → 掌握轨迹 + 学习速率 + 遗忘曲线
        ↓
TeachingAgent: 学习路径推荐 + 教学活动推荐 + 教学建议
        ↓
EvolutionAgent: 在线EM增量更新 + 参数漂移检测 + 因果发现 + 图谱演化
        ↓
教师反馈 → 参数更新 → 重新诊断 → 闭环
```

```
  │                    ↓

  │              教学决策 → 学习路径 → 学生分组 → 教学建议
  │
  ├─→ 知识追踪 → 掌握率轨迹 → 遗忘曲线 → 掌握时间预测
  │
  ├─→ Agent监控 → 5-Agent状态 → 消息流 → 事件日志

```

### 和"调用AI"的核心区别

| 环节 | 调用AI | AI原生（5-Agent框架） |
|------|--------|----------------------|
| 知识来源 | LLM解析教材 | KnowledgeAgent: 课件解析+课标对齐+EDC提取+语义对齐 |
| Q矩阵 | LLM分类 | DiagnosisAgent: 课标约束+NLP/LLM/手动三模式+数据联合估计 |
| 诊断 | LLM推理 | DiagnosisAgent: CDM统计推断(EM算法) |
| 根因 | LLM沿图谱猜 | DiagnosisAgent: CDM参数+图谱前驱链 |
| 追踪 | 无 | TracingAgent: BKT时序建模+遗忘曲线 |
| 教学 | LLM生成建议 | TeachingAgent: 图谱路径+模型参数驱动推荐 |
| 演化 | 边权重更新 | EvolutionAgent: 在线EM+漂移检测+因果发现+图谱演化 |
| 反馈学习 | few-shot注入prompt | EvolutionAgent: CDM参数增量更新 |
| 系统主动性 | 被动等待用户操作 | Agent事件驱动+主动通知+阈值监控+漂移检测 |

## 8. 边界情况与错误处理

| 场景 | 处理方式 |
|------|---------|
| CDM模型不收敛 | 降级到IRT模型，标记"诊断精度降低" |
| 学生数据<10人 | 不做CDM估计，只做描述性统计 |
| Q矩阵教师全部推翻 | 保存教师版本，NLP版本标记"被修正" |
| 知识点无前驱 | 标记为"基础知识点"，根因就是它本身 |
| 课件解析失败 | 标记失败页，提示教师手动补充 |
| 课件与课标无法对齐 | 保留为"自定义知识点"，不强制对齐 |
| 反事实预测样本不足 | 标记"预测不可靠"，不显示具体数值 |
| 教师修正根因为图谱外知识点 | 允许，提示"是否添加为新知识点" |
| Agent通知过多 | 同一事件24小时内只通知1次，合并同类通知 |
| 偏相关检验样本不足 | 控制变量后样本<20，标记"检验不可靠" |
| 参数漂移检测误报 | 连续2次检测到漂移才触发告警，单次标记"待观察" |
| BKT数据不足（<3次观测） | 标记"追踪数据不足"，只显示CDM快照不显示轨迹 |
| 跨项目数据引用 | 禁止，项目间严格数据隔离 |
| 6层层级缺失中间层 | 允许跳层，但标记"层级不完整"，图谱中虚线连接 |
| EDC提取关系冲突 | 保留置信度最高的关系，冲突关系标记"待确认" |
| 语义对齐失败 | 保留原始知识点名称，标记"未对齐"，允许教师手动关联 |
| Agent处理超时 | 标记任务"超时"，通知用户，保留上次成功结果 |
| 在线EM增量更新发散 | 回滚到上次稳定参数，标记"更新失败"，触发全量重估计 |

## 9. AI原生功能说明

> 以下功能是区分"AI产品"和"调用AI的产品"的关键。核心区别：系统有自己的模型、参数、推理规则和演化机制，LLM是工具不是引擎。

### 5-Agent协作框架（核心）

**是什么**：5个专业Agent协作完成从知识构建到诊断追踪到教学推荐到持续演化的全流程

**为什么是AI原生**：调用AI的产品是单次请求-响应，没有协作、没有状态、没有演化。5-Agent框架中，每个Agent有自己的职责、状态和触发条件，Agent之间通过消息传递协作，形成闭环。KnowledgeAgent构建图谱后通知DiagnosisAgent，DiagnosisAgent诊断后通知TracingAgent和TeachingAgent，EvolutionAgent检测到变化后通知DiagnosisAgent重新诊断——这是自主协作，不是被动调用。

### CDM认知诊断

**是什么**：用认知诊断模型（CDM）从学生答题数据中估计知识掌握状态

**为什么是AI原生**：这不是让LLM"分析一下错题"，而是用统计模型从数据中推断。CDM有自己的参数（slip、guess、掌握概率），有自己的推理规则（DINA：所有相关知识点都掌握才能答对），可以从数据中学习参数，可以用数学方法验证。

```
DINA规则: P(答对|知识状态α) = (1-slip) × ∏α_k + guess × (1-∏α_k)
其中: α_k = 1表示掌握知识点k, ∏α_k = 1当且仅当所有Q矩阵相关知识点都掌握
参数估计: EM算法, 从学生答题数据中估计slip, guess, α
```

### BKT知识追踪

**是什么**：用贝叶斯知识追踪（BKT）对学生知识状态进行时序建模

**为什么是AI原生**：LLM无法做时序预测——它没有时间概念。BKT有转移矩阵（P(L_t|L_{t-1})），可以从数据中估计学习速率和遗忘参数，可以预测未来掌握概率。这是参数化的时序模型，不是文本生成。

```
BKT四参数: P(L0)初始掌握概率, P(T)学习转移概率, P(G)猜对概率, P(S)失误概率
预测: P(L_t) = P(L_t|L_{t-1}) × P(L_{t-1}) + P(T) × (1 - P(L_{t-1}))
遗忘: P(L_t|L_{t-1}) 随时间衰减
```

### Q矩阵联合估计

**是什么**：题目-知识点映射不是LLM分类，而是从数据中联合估计

**为什么是AI原生**：LLM分类是"调用"，联合估计是"从数据中发现规则"。Q矩阵的每一项不是固定的，而是有概率的，可以从学生答题模式中验证和优化。

```
Step 1: 课标约束缩小候选集 → NLP/LLM在约束内选择
Step 2: 有学生数据后，用CDM联合估计验证Q矩阵
Step 3: 教师修正后，Q矩阵作为约束条件重新估计
```

### 在线EM增量更新

**是什么**：新数据到达后，CDM参数增量更新而非全量重估计

**为什么是AI原生**：调用AI的产品没有"更新"概念——每次都是重新调用。AI原生的产品有参数、有状态、有增量更新。在线EM是数学操作（充分统计量增量），不是文本操作（改prompt）。更新后所有后续诊断都基于新参数。

### 参数漂移检测

**是什么**：监控CDM/BKT参数是否发生统计显著变化

**为什么是AI原生**：调用AI的产品没有"漂移"概念——每次调用都是独立的。AI原生的产品有参数状态，可以检测参数是否偏离历史分布。漂移检测是统计检验（似然比检验/卡方检验），不是LLM判断。

### 因果发现

**是什么**：从数据中发现知识图谱中未定义的前驱关系

**为什么是AI原生**：传统产品的图谱是静态的。AI产品的图谱是活的——数据越多，关系越准确。偏相关检验比简单相关更接近因果，是统计方法不是LLM推理。

```
条件: 至少30个学生在A和B上都有数据
Step 1: 计算简单相关 P(B掌握|A掌握) - P(B掌握|A未掌握)
Step 2: 控制其他知识点后，计算A和B的偏相关系数
Step 3: 偏相关系数>0.3且p<0.05 → 建议A是B的前驱
Step 4: 教师确认后添加到图谱
```

### 课件优化反哺

**是什么**：诊断结果反哺课件，告诉教师"哪个知识点讲得不够"

**为什么是AI原生**：传统产品的数据流是单向的（课件→诊断）。AI产品的数据流是双向的（课件→诊断→课件优化）。这形成了一个闭环：课件质量影响诊断结果，诊断结果反过来指导课件改进。

## 10. Agent行为规格

### DiagnosisAgent — 诊断智能体

| 维度 | 规格 |
|------|------|
| **职责** | CDM参数估计、知识状态推断、根因追溯、Q矩阵管理 |
| **触发条件** | ① 收到KnowledgeAgent的图谱构建完成通知 ② 收到答题数据导入事件 ③ 收到EvolutionAgent的重新诊断通知 ④ 教师修正Q矩阵后 |
| **能力** | DINA模型EM参数估计、知识状态向量计算、沿前驱链根因追溯（最深层弱节点）、Q矩阵生成(NLP/LLM/手动)、Q矩阵校验、班级诊断聚合 |
| **输出** | 学生知识掌握状态向量、根因追溯路径(含置信度)、CDM模型参数(slip/guess/AIC/BIC)、Q矩阵(含校验结果)、班级诊断报告 |
| **协作关系** | ← 接收KnowledgeAgent图谱完成通知；→ 通知TracingAgent诊断完成；→ 通知TeachingAgent诊断完成；← 接收EvolutionAgent重新诊断通知 |
| **根因算法** | 对每个未掌握知识点u，沿前驱链遍历所有祖先节点，在所有P(掌握)<0.5的祖先中，选择深度最大且P值最低的节点作为根因。如果前驱链上所有节点P(掌握)≥0.5，根因就是u本身 |

| 维度 | 规格 |
|------|------|
| **职责** | 图谱构建、EDC提取、语义对齐、前驱发现、考点考法管理 |
| **触发条件** | ① 课件上传事件 ② 课标导入事件 ③ 收到EvolutionAgent的图谱演化通知 |

| **协作关系** | → 通知DiagnosisAgent图谱构建完成；← 接收EvolutionAgent图谱演化通知(新前驱关系) |

### TracingAgent — 追踪智能体

| 维度 | 规格 |
|------|------|
| **职责** | BKT预测、学习速率估计、遗忘曲线、掌握时间预测 |
| **触发条件** | ① 收到DiagnosisAgent诊断完成通知 ② 新答题数据到达(增量追踪) |
| **能力** | BKT四参数估计(P(L0)/P(T)/P(G)/P(S))、掌握轨迹时序预测、学习速率计算、遗忘曲线拟合(R(d,s)=e^(-d/s))、掌握时间预测(预计达标练习次数) |
| **输出** | 学生掌握轨迹(时间序列P(掌握)值)、学习速率(P(T))、遗忘参数和衰减曲线、预计掌握时间 |
| **协作关系** | ← 接收DiagnosisAgent诊断完成通知；→ 通知EvolutionAgent追踪结果(提供时序数据) |

### TeachingAgent — 教学智能体

| 维度 | 规格 |
|------|------|
| **职责** | 学习路径生成、活动推荐、教学建议、效果追踪 |
| **触发条件** | ① 收到DiagnosisAgent诊断完成通知 ② 教师请求教学建议 |
| **能力** | 基于图谱前驱链生成学习路径、基于CDM参数排序补课优先级、推荐教学活动(练习/讲解/演示)、生成教学建议(量化预测+自然语言)、效果追踪(建议采纳后跟踪掌握率变化) |
| **输出** | 学习路径(知识点序列+每步预测提升)、补课优先级列表(含量化预测)、教学活动推荐、教学建议(含模型参数依据)、效果追踪报告 |
| **协作关系** | ← 接收DiagnosisAgent诊断完成通知；→ 通知EvolutionAgent教学决策数据(提供反馈信号) |

### EvolutionAgent — 演化智能体

| 维度 | 规格 |
|------|------|
| **职责** | 在线EM增量更新、参数漂移检测、因果发现、图谱演化 |
| **触发条件** | ① 收到TracingAgent追踪结果 ② 收到TeachingAgent教学决策数据 ③ 新答题数据到达 ④ 定时巡检(参数漂移检测) |
| **能力** | 在线EM增量更新CDM参数(θ_{t+1}=(1-η)·θ_t+η·M-step(E-step(x_t,θ_t)))、参数漂移检测(drift=\|θ_t-θ_{t-W}\|/σ_{t-W}，>3σ触发告警)、因果发现(偏相关检验+PC-Stable)、图谱演化(新前驱关系建议)、增量更新发散检测与回滚 |
| **输出** | 增量更新后的CDM/BKT参数、漂移检测报告(参数变化量+是否显著)、新发现的前驱关系(含偏相关系数+p值)、图谱演化建议 |
| **协作关系** | ← 接收TracingAgent追踪结果；← 接收TeachingAgent教学决策数据；→ 通知DiagnosisAgent重新诊断(参数更新/漂移检测)；→ 通知KnowledgeAgent图谱演化(新前驱关系) |

### Agent协作时序

```
时间轴 →

T1: 教师上传课件
    KnowledgeAgent: 解析课件 → 构建图谱 → 通知DiagnosisAgent

T2: 教师导入答题数据
    DiagnosisAgent: Q矩阵生成 → CDM参数估计 → 根因追溯
    DiagnosisAgent: 通知TracingAgent + TeachingAgent

T3: 诊断完成
    TracingAgent: BKT时序建模 → 掌握轨迹 → 通知EvolutionAgent
    TeachingAgent: 学习路径 → 教学建议 → 通知EvolutionAgent

T4: 演化
    EvolutionAgent: 增量更新参数 → 漂移检测 → 因果发现
    EvolutionAgent: 如有显著变化 → 通知DiagnosisAgent重新诊断
    EvolutionAgent: 如发现新前驱 → 通知KnowledgeAgent图谱演化

T5: 闭环
    DiagnosisAgent: 基于更新后的参数重新诊断
    (回到T3)
```

| drift_detected | EvolutionAgent | 通知 | 参数漂移告警 |

模板检测规则：≥80%的页面包含"知识点"和"重点"区域标识时判定为模板PPT。
