文件：app/services/dina_model.py

组件说明
响应概率计算P(Rij=1 | αi) = (1-sj)ηij × gj1-ηij
其中 ηij = ∏k αikqjk 为理想响应
EM算法·E步计算每个学生属于每个可能知识状态的后验概率 P(α | R)，基于当前模型参数

第七章 核心算法实现

7.1 DINA模型响应概率

DINA模型定义学生 i 答对题目 j 的概率为：

P(Rij = 1 | αi) = (1 − sj)ηij · gj1 − ηij

其中 ηij = Πk=1K αikqjk，当学生掌握了题目考核的所有知识点时 η=1（反映为"理想掌握"），否则 η=0。

7.2 EM参数估计算法

伪代码：

输入: Q矩阵(M×K), 答题矩阵R(S×M), 最大迭代I_max
输出: slip向量(s₁,...,s_M), guess向量(g₁,...,g_M), alpha后验

初始化: sⱼ=0.2, gⱼ=0.2, 状态先验均匀分布

for iter in 1..I_max:
    # E步：计算后验
    for each student i:
        for each 可能状态 α ∈ {0,1}^K:
            计算 P(α | Rᵢ) = P(Rᵢ | α) × P(α) / Σ P(Rᵢ | α′) × P(α′)

    # 基于后验计算期望充分统计量
    for each question j:
        nⱼ₀₀ = Σᵢ Σ_{α:η=0} P(α|Rᵢ) × (1-Rᵢⱼ)  # η=0且答错
        nⱼ₀₁ = Σᵢ Σ_{α:η=0} P(α|Rᵢ) × Rᵢⱼ       # η=0且答对
        
    # M步：更新参数
    gⱼ = nⱼ₀₁ / (nⱼ₀₀ + nⱼ₀₁)
    sⱼ = 1 - nⱼ₁₁ / (nⱼ₁₀ + nⱼ₁₁)

    if max|param_new - param_old| < 1e-6: break

7.3 蒙特卡洛EM（n_kp > 15）

当K>15时，2K超过32768，枚举所有状态不可行。降级策略：

  • 从历史alpha后验分布中采样M=500个代表性状态
  • 用采样状态近似E步积分：P(α | R) ≈ P(R | α) / Σα′∈Sample P(R | α′)
  • 降低迭代次数上限至5轮，平衡精度和性能

7.4 根因追溯算法

输入：学生i的alpha向量、知识图谱G、薄弱阈值θ_w（默认0.4）

对于每个 P_mastery(ik) < θ_w 的薄弱知识点 k:
    candidates = []
    沿G中的前驱链追溯：
        对每个前驱节点 p：
            score_p = depth(p) × (1 - P_mastery(ip))
            candidates.append({node: p, score: score_p})
    
    root_cause = argmax(candidates.score)  # 最深×最薄弱
    构建从 root_cause 到 k 的追溯路径树
    
返回：每个薄弱知识点的根因节点 + 路径

7.5 反事实推理

基于CDM参数化模型：假设学生i的知识状态从 αi 变为 α′i（补上某个知识点），重新计算所有知识点的P(掌握)，量化"如果补上X，Y的掌握率将从A%变为B%"。

7.6 遗忘曲线

基于Ebbinghaus指数衰减模型：M(t) = M0 · e−λt

  • M₀：初始掌握率（诊断时）
  • λ：衰减率参数（由历史数据拟合）
  • 半衰期 t1/2 = ln(2) / λ（推荐复习时间）

  • 注册机制：每个Agent在初始化时向Bus注册，声明订阅的消息类型
  • 消息路由：Agent发送消息时指定类型，Bus根据注册表路由给订阅方
  • 事件日志：所有消息均持久化到 agent_events 表，用于监控和审计

8.3 Agent生命周期

  1. 初始化：Agent实例化 → 加载配置 → 订阅消息 → 设置事件处理器
  2. 运行：监听消息 → 处理业务逻辑 → 发送结果消息
  3. 空闲：无任务时进入等待状态
  4. 异常：处理失败时记录错误日志，触发告警通知
  5. 关闭：收到关闭信号 → 完成当前任务 → 保存状态 → 取消订阅
