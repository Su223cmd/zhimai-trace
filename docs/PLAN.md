## 目标架构：4层+5-Agent框架

```
┌─────────────────────────────────────────────────────────────────┐
│  第1层：前端 (React 18 + TypeScript)                             │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐                  │
│  │知识总览 │ │诊断详情 │ │教学决策 │ │课件管理 │                  │
│  └────────┘ └────────┘ └────────┘ └────────┘                  │
│  Ant Design 5 │ AntV G6 │ ECharts                              │
├─────────────────────────────────────────────────────────────────┤
│  第2层：API层 (FastAPI)                                          │
│  /api/courseware  /api/knowledge  /api/homework                 │
│  /api/diagnosis   /api/curriculum /api/learning-path            │
│  /api/agent       /api/auth                                     │
├─────────────────────────────────────────────────────────────────┤
│  第3层：5-Agent协作框架                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │DiagnosisAgent │  │KnowledgeAgent│  │ TracingAgent │          │
│  │ DINA+根因+Q矩阵│  │ 图谱+课标+课件│  │ BKT+速率+遗忘│          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
│  ┌──────────────┐  ┌──────────────┐                             │
│  │TeachingAgent │  │EvolutionAgent│                             │
│  │ 路径+教学+分组 │  │ OnlineEM+因果 │                             │
│  └──────────────┘  └──────────────┘                             │
│  ┌──────────────────────────────────────────────┐               │
│  │ AgentBus 消息总线 (send/receive/broadcast)    │               │
│  └──────────────────────────────────────────────┘               │
├─────────────────────────────────────────────────────────────────┤
│  第4层：数据层 (PostgreSQL + 文件系统)                            │
│  DB为唯一数据源 │ 课标JSON外置 │ 启动时加载到内存                  │
└─────────────────────────────────────────────────────────────────┘
```

**5-Agent消息流**：

```
课件上传 ──→ KnowledgeAgent ──→ AgentBus ──→ DiagnosisAgent
                                              │
答题导入 ──→ DiagnosisAgent ──→ AgentBus ──→ TracingAgent
                                              │
                                    AgentBus ──→ TeachingAgent
                                              │
新数据   ──→ EvolutionAgent ──→ AgentBus ──→ DiagnosisAgent (循环)
```

| Agent | 封装的原有服务 | 触发条件 | 输出消息 |
|-------|-------------|---------|---------|
| DiagnosisAgent | dina_model + root_cause + qmatrix_service | 收到答题数据/因果图更新 | `diagnosis_completed` |
| KnowledgeAgent | knowledge_service + curriculum_service + courseware_service | 课件上传/课标导入 | `graph_updated` |
| TracingAgent | knowledge_tracing (BKT) + 遗忘曲线 | 收到`diagnosis_completed` | `tracing_updated` |
| TeachingAgent | learning_path + teaching + class_service | 收到`diagnosis_completed` | `teaching_plan_created` |
| EvolutionAgent | online_em + causal_discovery | 新数据到达/定期触发 | `params_updated` / `causal_graph_updated` |

**根因追溯bug修复说明**：

当前实现沿前驱链追溯时，返回第一个P(掌握)<0.5的节点。正确行为应返回前驱链上P(掌握)最低的最深层节点——即真正的"根因"而非"最近的弱节点"。

算法修正：
1. 对每个未掌握知识点u，沿前驱链遍历所有祖先节点
2. 在所有P(掌握)<0.5的祖先中，选择深度最大且P值最低的节点作为根因
3. 如果前驱链上所有节点P(掌握)≥0.5，根因就是u本身

## 执行规则

1. **Phase 1是硬前置**：任何后续Phase不得在Phase 1未通过验收时启动
2. **验收标准不可降级**：每个Phase的验收标准必须全部通过，不允许"部分通过"
3. **问题不跨Phase**：在当前Phase发现的问题必须在当前Phase修复
4. **代码风格一致**：遵循项目现有代码风格，不添加注释（除非被要求）
5. **CDM核心原则**：所有诊断计算必须用NumPy/SciPy，绝不调用LLM做诊断
6. **LLM使用原则**：LLM只用于文本理解（课件解析、知识点匹配、自然语言生成），不用于诊断推理
7. **提交规范**：每个Phase完成后commit一次，commit message格式：`phase{N}: {摘要}`
